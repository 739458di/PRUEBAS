Por que existe el buffer?

Problema: Los usuarios envian multiples mensajes rapidos:

[09:01:00] "Hola" [09:01:01] "quiero info" [09:01:02] "del Mazda 3"

Solucion: Buffer espera 1.5s despues del ULTIMO mensaje:

09:01:00 -> buffer: ["Hola"] (timer 1.5s) 09:01:01 -> buffer: ["Hola","quiero info"] (reset timer) 09:01:02 -> buffer: ["Hola","quiero info","del Mazda 3"] (reset) 09:01:03.5 -> timer fire! processBuffered() texto = "Hola quiero info del Mazda 3"

Resultado: Un solo mensaje combinado al AI en vez de 3 respuestas separadas.

bufferKey = jid del contacto (ej: 5218120066355@s.whatsapp.net). Cada contacto tiene su propio buffer independiente.

Sistema Anti-Crash - Triple Failsafe

Failsafe #1: ANTI-CRASH WRAPPER

ACTIVO

try/catch alrededor de TODO el cuerpo de processBuffered

6044 try { // ANTI-CRASH ... // 280 lineas de logica 6323 } catch(_pbFatalErr) { // Log error + stack // Save msgs to DB // Release locks }

Si processBuffered crashea: mensajes se guardan a DB con 'crash-save-*' para que recovery los encuentre

Failsafe #2: Per-message catch

ACTIVO

Cada mensaje individual en messages.upsert tiene su propio try/catch

for (msg of messages) { try { // process this msg } catch(e) { // skip, continue next } }

Un mensaje malo no mata el procesamiento de los demas

Failsafe #3: Process handlers

ACTIVO

Handlers globales previenen que el proceso Node.js muera

7200 process.on('uncaughtException', (err) => { // Log pero NO exit // Memory cleanup si critico }); 7210 process.on('unhandledRejection', (reason) => { // Log pero NO crash });

El proceso nunca se mata solo. PM2 reinicia si es necesario.
